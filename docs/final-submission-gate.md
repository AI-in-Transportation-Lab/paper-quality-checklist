# Final Submission Gate

Run this page after all writing, analysis, formatting, and coauthor review are complete.

## Venue and Files

- [ ] Target venue name, track, article type, and deadline are correct.
- [ ] The manuscript follows the current official template.
- [ ] Page limit, word limit, figure limit, supplement limit, and file-size limit are satisfied.
- [ ] Blind-review or single-blind requirements are handled correctly.
- [ ] Required checklists, cover letter, highlights, graphical abstract, data statement, or ethics statement are ready.
- [ ] Supplemental files are named clearly and referenced from the manuscript.

## Authorship and Administration

- [ ] Author order is final.
- [ ] Names, affiliations, emails, ORCID IDs, and corresponding author are correct.
- [ ] Funding, acknowledgments, conflicts of interest, and data availability statements are complete.
- [ ] Coauthors approved the exact final version.
- [ ] Permissions are cleared for reused figures, tables, datasets, and software.

## Manuscript Integrity

- [ ] Title, abstract, research questions, results, and conclusion align.
- [ ] Every major claim has direct support.
- [ ] No unsupported causal, safety, policy, or generalization claims remain.
- [ ] Limitations are specific and honest.
- [ ] AI-use disclosure matches venue policy.
- [ ] No hidden text, reviewer instructions, prompt-injection text, or private comments remain.

## Figures, Tables, Equations, and References

- [ ] Every figure, table, equation, algorithm, appendix, and supplement is cited in order.
- [ ] Figure and table captions are complete.
- [ ] Fonts, lines, symbols, colors, and labels are readable in the final PDF.
- [ ] Units, sample sizes, and uncertainty are visible where needed.
- [ ] Equations compile correctly and all variables are defined.
- [ ] Zotero or reference-manager output was checked manually.
- [ ] Every reference is real, complete, and cited in the text.

## Statistics, Data, and Reproducibility

- [ ] Statistical tests and models match the data structure.
- [ ] Assumptions, diagnostics, effect sizes, and uncertainty are reported.
- [ ] Data source, preprocessing, exclusions, and missing data are documented.
- [ ] Code, data, prompts, models, seeds, environment, and restrictions are stated.
- [ ] Tables and figures can be regenerated from the final analysis workflow.
- [ ] Any restricted data or code has a clear access explanation.

## Final PDF Review

- [ ] Compile the final PDF from a clean source.
- [ ] Search the PDF for `error`, `??`, `undefined`, `TODO`, `fix`, `citation needed`, and unresolved comments.
- [ ] Open the PDF in a separate viewer and inspect every page.
- [ ] Check page breaks, margins, equations, captions, references, appendices, and supplements.
- [ ] Confirm the PDF uploaded to the submission system matches the approved final version.
- [ ] Download the submitted proof or confirmation copy and archive it with the submission record.
